Package lev (Logging of Environment and Version information)

Author: Herb Ward.

All the functions discussed here are for use only by SOFI personnel.

This package is for putting version information (on modules, tables, etc) and environment information (date, user, machine, etc) into the hierarchical data structure so that this info can be archived with the output. Thus, should an incompatibility be discovered later, the affected portions of our processed-data archives (expensive in terms of CPU power consumption) can be isolated and deleted.

Below is the API (functions intended for calling from outside the lev package). The functions are listed in the approximate order that they should be called. All these functions return FALSE in case of error, and TRUE otherwise. 

boolean levAddEnvironmentTable(DS_DATASET_T **ppEnv);
  /* Information on ppEnv:
  **      Read and write.
  **      INPUT(NON-NULL) dsAddTable to dataset 
  **      INPUT(NULL)     dsNewTable
  **      OUTPUT          pointer to new table
  */
boolean levAddVersionTable(DS_DATASET_T **ppVer);
  /* Information on ppVer:
  **      Read and write.
  **      INPUT(NON-NULL) dsAddTable to dataset 
  **      INPUT(NULL)     dsNewTable
  **      OUTPUT          pointer to new table
  */
boolean levFindEnvironmentTable(DS_DATASET_T **ppEnv);
  /* Information on ppEnv: 
  **      Write only.
  **      OUTPUT   point to newest environ table
  */
boolean levFindVersionTable(DS_DATASET_T **ppVer);
  /* Information on ppVer: 
  **      Write only.
  **      OUTPUT   point to newest version table
  */
boolean levRegisterEnvironment(DS_DATASET_T *pEnv,
          const char *comment);
  /* pEnv must point to a table, and you can get pEnv
  ** either from output of levAddEnvironmentTable() or 
  ** levFindEnvironmentTable().
  */

          enum type,char *ver);
  /* pVer must point to a table, and you can get pVer
